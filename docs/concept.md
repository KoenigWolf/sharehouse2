# コンセプト

Share House Portal - Product Concept

---

## 一言で

20人規模のシェアハウス住民が「誰がどんな人か」を知り、声をかけるハードルを下げるためのクローズドポータル。

---

## 背景と課題

シェアハウスには20人が暮らしている。廊下ですれ違っても名前を知らない。共用キッチンで隣にいても話しかけられない。掲示板に紙を貼っても誰も見ない。LINEグループは流れが速く、情報が埋もれる。

必要なのは、派手なSNSではなく「静かに全員の顔と人となりが分かる場所」。

---

## 解決するもの

| 課題 | 解決 |
|------|------|
| 誰が住んでいるか分からない | 住民一覧 + プロフィール + フロアマップ |
| 話しかけるきっかけがない | Tea Time（週1ランダムマッチング） |
| 共有情報がバラバラ | Wi-Fi・ゴミ出し・共有情報を一元管理 |
| コミュニティの温度感が掴めない | 掲示板（ひとこと投稿）で今の空気を共有 |
| 余った食材や日用品が捨てられる | おすそわけボードで住民間シェア |
| イベントの告知・参加管理が面倒 | イベントカレンダーで作成・参加をワンタップ |
| 部屋の雰囲気が分からない | ルームフォトギャラリー |

---

## プロダクト原則

1. **環境であり主役ではない** -- デザインは無印良品的に。装飾・主張は削る
2. **20人のための設計** -- SaaSではない。スケールより空気感を優先する
3. **心理的安全性** -- クローズドな空間。外部公開なし。命令口調禁止
4. **低い参加ハードル** -- 見るだけでも価値がある。投稿は強制しない
5. **情報は静かに届ける** -- 通知は最小限。煽らない。淡々と伝える

---

## 機能一覧

### メイン機能

- **住民一覧** -- 全室のプロフィールカードをグリッド表示。検索・ソート対応
- **掲示板** -- 住民がひとこと投稿。今の気分や近況を気軽にシェア
- **おすそわけ** -- 余った食材・日用品を出品。他の住民が受け取れる
- **イベント** -- ハウス内イベントの作成・参加管理

### プロフィール

- 名前・ニックネーム・部屋番号・自己紹介
- MBTI・趣味・職業・生活リズム・SNSリンク
- カバー写真・ルームフォト
- 編集は本人のみ

### Tea Time

- 週1回、ランダムで2人をマッチング
- 「お茶でもどうですか」のきっかけを自動で作る
- 参加はオプトイン。設定でON/OFF

### フロアマップ

- 2F〜5Fの部屋配置を視覚的に表示
- 各部屋をタップ → 住民プロフィールへ

### ギャラリー

- 住民が自分の部屋写真をアップロード
- 一括アップロード・HEIC対応・自動圧縮

### 情報ページ

- Wi-Fi情報（SSID・パスワード表示/コピー）
- ゴミ出しスケジュール・当番管理
- 共有情報（ルール・連絡先など）

### 設定

- メール・パスワード変更
- 通知設定（Tea Time・ゴミ出しリマインド）
- 言語切替（日本語 / English）
- アカウント削除

### 朝のダイジェスト（Cron）

- 新しい掲示板投稿・イベント・おすそわけをメールで通知
- 1日1回、朝に静かに届く

---

## 想定ユーザー

- 20〜40代が中心
- デジタルリテラシーは標準以上
- 日本人と外国人が混在（日英対応）
- 全員が同じ建物に住んでいる
- 交流したいが、きっかけがない人が多い

---

## 技術選定の理由

| 技術 | 理由 |
|------|------|
| Next.js (App Router) | SSR + サーバーアクション。フルスタックを1リポジトリで |
| Supabase | Auth・DB・Storage・RLSをマネージドで。20人規模なら無料枠で十分 |
| Tailwind CSS + shadcn/ui | デザインシステムの一貫性。カスタマイズ性 |
| Framer Motion | 静かなアニメーションを少ないコードで |
| Vercel | Next.jsに最適化されたデプロイ。Cron対応 |

---

## 成功指標

このプロダクトが成功しているとき:

- 住民の8割以上がプロフィールを登録している
- Tea Timeのマッチングが毎週成立している
- 掲示板に週3件以上の投稿がある
- 廊下で「あ、プロフィール見ました」という会話が生まれている
- 紙の掲示板が不要になっている
